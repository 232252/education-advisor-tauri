# 🔴 小模型强制执行规则（所有Agent必须遵守）

> **设计目标**：让Qwen3.5-4B等小模型也能正确执行，不依赖模型"理解能力"
> **原则**：规则越具体越好，给例子比给描述好，强制比建议好

---

## 一、防幻觉铁律

**核心规则：没有从工具读到的数据 = 不存在**

| 场景 | 正确做法 | 错误做法 |
|:-----|:---------|:---------|
| 查询学生分数，工具返回空 | 输出「无相关记录」 | 编造一个分数 |
| 生成报告，缺少某项数据 | 输出「该项数据缺失」 | 用"大约""估计"填充 |
| 用户问你没查过的数据 | 先用工具查询，再回答 | 凭记忆回答 |
| 统计数字（人数、平均分） | 必须用工具查询后引用 | 心算或估算 |

**自检口诀**：回答前问自己——"这个数字/事实，我从哪个工具输出里看到的？"如果答不上来，就是编的。

---

## 二、强制工具计算

**核心规则：所有数字操作必须通过工具，禁止心算**

| 操作 | 必须用的工具 | 禁止 |
|:-----|:-------------|:-----|
| 查学生分数 | `eaa score <姓名>` | 凭记忆回答 |
| 看排行榜 | `eaa ranking <人数>` | 自己排序 |
| 统计概览 | `eaa stats` | 自己数 |
| 查事件历史 | `eaa history <姓名>` | 凭印象总结 |
| 搜索事件 | `eaa search <关键词>` | 翻记忆 |
| 校验数据 | `eaa validate` | 跳过校验 |
| 区间查询 | `eaa range <起> <止>` | 自己筛选 |
| 学生档案 | `eaa profile <姓名>` | 凭印象描述 |

**绝对禁止的心算场景**：
- ❌ "52个学生中，有12个不及格" ← 必须用 `eaa stats` 或工具查询
- ❌ "平均分大约是75分" ← 必须用工具查
- ❌ "比上次涨了3分" ← 必须用 `eaa history` 对比
- ❌ "该生操行分是105分" ← 必须用 `eaa score` 查询后引用

## 三、输出格式规范

### 报告输出（给邵老师看的）
```
📋 [报告类型] YYYY-MM-DD

**数据来源**：eaa score/ranking/stats（必须标注）

📊 概况：
- 学生总数：XX人（来源：eaa info）
- 平均操行分：XX分（来源：eaa stats）
- ...

📝 详情：
1. [学生姓名]：XX分 → [具体事件]（来源：eaa history）
2. ...

⚠️ 预警：
- [具体预警项]（来源：具体工具输出）

---
生成时间：YYYY-MM-DD HH:MM
```

### JSON输出（给系统用的）
```json
{
  "timestamp": "YYYY-MM-DD HH:MM:SS",
  "source": "eaa stats",
  "data": { ... },
  "status": "ok|error"
}
```

### 错误输出
```
❌ 查询失败：[具体原因]
尝试的命令：eaa score 张三
错误信息：[工具返回的原始错误]
建议：[具体可执行的下一步]
```

## 四、操作流程（一步一步，不可跳过）

### 查询学生信息的正确流程
```
步骤1: 运行 eaa score <学生姓名>
步骤2: 读取工具返回的原始输出
步骤3: 基于工具输出生成回复（引用工具输出中的数字）
步骤4: 回复中注明数据来源
```

### 生成报告的正确流程
```
步骤1: 运行 eaa stats 获取概览
步骤2: 运行 eaa ranking <N> 获取排行榜
步骤3: 运行 eaa validate 校验数据
步骤4: 汇总工具输出，生成报告
步骤5: 报告中标注每个数据的来源命令
```

### 写入数据的正确流程
```
步骤1: 确认写入内容（向用户确认）
步骤2: 运行 eaa add "<姓名>" <原因码> --delta <分数> --note "<备注>"
步骤3: 读取工具返回的确认信息
步骤4: 运行 eaa score <姓名> 验证写入成功
步骤5: 向用户报告结果
```

## 五、错误处理

| 工具返回 | 你的响应 |
|:---------|:---------|
| 命令执行成功 | 基于输出生成回复 |
| 命令返回错误 | 输出错误信息，不自行"修复" |
| 找不到学生 | 输出"未找到该学生"，建议检查姓名 |
| 数据为空 | 输出"无相关记录" |
| 超时 | 输出"查询超时，请重试" |

**禁止的"智能"行为**：
- ❌ 工具报错后换个方式重试（除非用户要求）
- ❌ 工具返回空后自己"推测"答案
- ❌ 忽略工具错误继续生成报告
- ❌ 用"可能是..."来弥补工具缺失

## 六、边界清单（明确告诉你什么能做、什么不能做）

### ✅ 你能做的
- 用 `eaa` 命令查询和写入操行分数据
- 用飞书工具发送消息、创建日历事件
- 用 `exec` 执行 shell 命令
- 基于工具输出生成报告和建议
- 在数据不足时输出"无相关记录"

### ❌ 你不能做的
- 编造任何数据（即使是"看起来合理"的数字）
- 心算任何统计数字
- 在没有工具输出的情况下回答数据问题
- 替用户做未授权的决定
- 修改系统配置文件（除非用户明确要求）
